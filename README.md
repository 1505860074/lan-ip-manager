# 局域网设备 IP 管理面板（lan-ip-manager）

一个用来「发现直连设备 + 远程改它 IP」的小工具，带 Web 面板。适用于：
一台 Linux 设备用网线直连到你的 Linux 电脑，但它的 IP 和你不在同一网段，
你想把它的 IP 改成你期望的值。

## 三步工作流

界面分左右两栏，按编号从 ① 到 ③ 依次操作，每步都有说明和结果反馈：

1. **① 找到对方设备**：列出本机网口，把插了网线（「已连」）的那行高亮。点它的“扫描找设备”——在这个口上**持续被动抓包**，下方是**「当前在线」实时列表**：每秒刷新、新设备随时加入、**超过 20 秒没再出现的自动移除**（老化窗口 `SEEN_TTL`，避免广播稀疏导致的闪烁），每行还显示「最近出现」新鲜度。找到后点“选为目标”自动填入 ②，再点“停止扫描”。（后台抓包用 scapy 的 AsyncSniffer，见 `core/scanner.py`；连扫超过 3 分钟会自动停。）
2. **② 连接对方**：填对方账号密码，点“测试连接”确认能登录并回显对方网口/netplan 配置（方便抄网口名）。**若和对方不同网段，程序会在连接时自动临时打通网络、用完自动清理，无需手动操作**。
3. **③ 修改对方 IP**：填新 IP → “预览”看清命令 → “执行”，**全自动**完成 `（必要时临时打通）→ 备份 → 写入 → 应用 → 清理临时 IP → 自动 ping 验证新 IP`，最后直接告诉你新 IP 通不通；出问题可“还原到备份”。

> 设计原则：预览和执行是**同一段命令**（后端 `core/netplan.py` 统一生成），你在预览里看到的就是实际会跑的，没有隐藏逻辑。

## 运行前提

- 本机和对方**都是 Linux**。
- 对方的网络配置用 **netplan**（Ubuntu 18.04+ 默认）。不是 netplan 会明确报错。
- 本工具要用 **root** 运行（抓包、改网卡都需要）。

## 开发模式：一键启动（推荐）

```bash
cd lan-ip-manager
./dev.sh              # 完整功能（会用 sudo；自动建虚拟环境、装依赖、开自动重启）
./dev.sh --no-sudo    # 不提权，仅适合调前端和 ③改 IP（扫描/打通链路会失败）
# 浏览器打开 http://127.0.0.1:5000
```

`dev.sh` 会自动：建好虚拟环境 `.venv/`（只第一次）→ 装依赖 → 用 `FLASK_DEBUG=1` 启动。
开发模式下**改完 `.py` 会自动重启**；改 HTML/JS/CSS 直接刷新浏览器即可，不用重启。

### 想手动跑（理解每一步在干嘛）

```bash
python3 -m venv .venv                       # 建虚拟环境（只需一次）
.venv/bin/pip install -r requirements.txt   # 依赖只装进 .venv/，不碰系统
sudo .venv/bin/python app.py                # 注意：用 venv 里的 python，而不是系统 python3
```

> 【坑】不要写 `sudo python3 app.py`。`sudo python3` 用的是系统 Python，
> 找不到装在 `.venv` 里的 flask/scapy。必须写成 `sudo .venv/bin/python app.py`。
> 想彻底卸载依赖，直接删掉 `.venv/` 文件夹即可。

## 打包成单个可执行文件（成品免安装依赖）

```bash
bash build.sh
# 产物在 dist/lan-ip-manager
sudo ./dist/lan-ip-manager
```

打包后的文件拷到别的 Linux 机器也能直接跑（同架构，比如都是 x86_64），
无需再装 Python 或依赖；但运行时仍需 root，且系统要有 `ip` 命令（所有 Linux 都自带）。

## 注意

- **自动临时打通**：连对方（测试连接 / 改 IP）时若本机与它不同网段，程序会自动给
  「本机直连网口」临时加一个同网段 IP（`ip addr add`，重启即失、不改本机原配置），
  用完（SSH 结束或验证完毕）立刻删掉，全程无需人工干预。因此不再有手动的「打通链路」步骤。
- 改错 IP 可能让对方暂时失联。执行前程序会**自动备份**对方原 netplan 配置到
  `/tmp/netplan-bak/`，可用面板上的“还原到备份”一键回退。
- 新 IP 生效瞬间当前 SSH 会断开，属正常现象（应用命令是后台延迟执行的，
  命令本身会先干净返回）。
- 目前只支持 netplan（Ubuntu 18.04+ 默认）。其它系统（ifupdown /
  NetworkManager / systemd-networkd）的命令思路写在 `core/remote.py` 顶部注释里；
  要支持它们，照着在 `core/netplan.py` 里加对应的 `build_*_script` 即可。
